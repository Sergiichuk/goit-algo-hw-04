# goit-algo-hw-04
home work 04

У цій роботі я порівняв три алгоритми сортування:
Сортування вставками,
Сортування злиттям,
Timsort (це той алгоритм, який використовує Python у функції sorted()).

Метою було перевірити, наскільки вони відрізняються за швидкістю на масивах різного розміру та підтвердити теоретичні оцінки часу роботи.




Я створив масиви випадкових чисел розміру:

500 елементів / 2000 елементів / 5000 елементів.
Для вимірювання часу використав модуль timeit, який точно заміряє скільки працює певний фрагмент коду.




Приблизно виходить так:

Розмір масиву: 500/
Insertion sort: ~0.05 сек.
Merge sort:     ~0.01 сек.
Timsort:        ~0.0007 сек.

Розмір масиву: 2000/
Insertion sort: ~0.8 сек.
Merge sort:     ~0.03 сек.
Timsort:        ~0.002 сек.

Розмір масиву: 5000/
Insertion sort: ~4 сек.
Merge sort:     ~0.10 сек.
Timsort:        ~0.004 сек.



Висновки:

Insertion sort виявився найповільнішим, бо він має складність O(n²).
Merge sort значно швидший, бо працює за O(n log n).
Timsort показав найкращі результати.
Він швидкий, бо в ньому поєднано:

вставки (для маленьких шматків), злиття (для великих), а також він використовує вже впорядковані ділянки масиву.
Через це вбудований sorted() у Python працює краще, ніж більшість алгоритмів, які можна написати вручну.
